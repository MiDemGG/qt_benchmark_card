# Qt Benchmark System

Кроссплатформенное приложение на C++/Qt, предназначенное для комплексного тестирования производительности вычислительной подсистемы, дисковых операций и графического вывода.  
Система не только замеряет скорость текущей машины, но и сравнивает её с эталонными показателями других комплексов (ПАК1 и ПАК2).

## 📊 Методология тестирования

Программа проводит три независимых теста, результаты которых объединяются в интегральный показатель производительности с помощью среднего геометрического.

1.  **Calc (Вычислительная мощность)**: Циклическое возведение числа в степень (8 000 000 итераций) с использованием `std::pow`. Проверяет скорость работы CPU с числами с плавающей запятой.
2.  **Write (Дисковые операции)**: Копирование медиафайла (`video.mp4`) средствами `std::filesystem`. Оценивает пропускную способность файловой системы и накопителя.
3.  **Draw (Графическая подсистема)**: Отрисовка серии примитивов (эллипсов) с использованием `QPainter` и имитацией задержки отрисовки кадра.

## ✨ Особенности реализации

*   **Автоматический запуск**: Тесты стартуют автоматически через 500 мс после инициализации окна.
*   **Сравнительный анализ**: Система выводит две таблицы данных:
    *   **Сырые данные**: реальное время выполнения тестов в секундах.
    *   **Нормированные данные**: относительные коэффициенты производительности, рассчитанные по отношению к эталонному комплексу ПАК2.
*   **Визуальный контроль**: Во время графического теста на экране отрисовываются зеленые индикаторы (круги), подтверждающие работу графического движка.
*   **Форматированный вывод**: Использование моноширинного шрифта (Courier New) для обеспечения идеального выравнивания колонок в отчете.

## 🛠 Технологический стек

*   **C++17/20**: Активное использование библиотек `<chrono>` для высокоточных замеров и `<filesystem>` для работы с файлами.
*   **Qt 6**: Использование модулей `Widgets` и `Gui`.
*   **CMake**: Система автоматизированной сборки и развертывания (Deployment) приложения.

## 🔧 Структура файлов

*   `mainwindow.cpp` / `mainwindow.h`: Ядро системы — реализация алгоритмов тестирования и логики расчетов.
*   `mainwindow.ui`: Дизайн главного окна с настроенной цветовой палитрой и элементами интерфейса.
*   `CMakeLists.txt`: Скрипт сборки с поддержкой автоматической подготовки дистрибутива приложения.

## 🚦 Как запустить

1.  Убедитесь, что в рабочей директории приложения находится файл `video.mp4` (необходим для теста записи).
2.  Соберите проект через CMake и запустите исполняемый файл.
3.  Дождитесь завершения автоматического цикла тестов. Результаты появятся в текстовом поле в нижней части окна.

---
*Интегральный показатель (I) рассчитывается как среднее геометрическое нормализованных значений, что позволяет объективно оценить сбалансированность системы.*
